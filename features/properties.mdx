## Overview

Properties are central to real estate CRM. Relio provides comprehensive property tracking with parcel mapping, ownership data, sale history, and automatic enrichment from county assessor records.

## Creating Properties

<Steps>
  <Step title="Enter property address">
    The only required field:
    - `address` - Street address
    
    Optional location fields:
    - `city`, `state`, `zip`, `country`
    - `latitude`, `longitude` - Auto-populated from geocoding
  </Step>
  
  <Step title="Set property type and status">
    Classify the property:
    - `propertyType` - `residential`, `commercial`, `land`, `industrial`, `multi_family`
    - `status` - `active`, `under_contract`, `sold`, `off_market`, `for_lease`
  </Step>
  
  <Step title="Add property characteristics">
    Basic property details:
    - `bedrooms` - Number of bedrooms
    - `bathrooms` - Number of bathrooms (decimal, e.g., 2.5)
    - `sqft` - Interior square footage
    - `lotSqft` - Lot size in square feet
    - `units` - Number of units (for multi-family)
    - `yearBuilt` - Year of construction
  </Step>
  
  <Step title="Enter pricing and financial data">
    All monetary values stored in cents:
    - `listPrice` - Current listing price
    - `soldPrice` - Final sale price
    - `estimatedValue` - Estimated market value
    - `equity` - Equity percentage (0-100+)
    - `capRate` - Capitalization rate percentage
  </Step>
  
  <Step title="Add parcel identifiers">
    Link to county records:
    - `parcelId` - County parcel ID
    - `apn` - Assessor's Parcel Number
    - `llUuid` - Regrid universal parcel identifier
    - `mlsId` - MLS listing number
  </Step>
</Steps>

<Info>
  When you provide a parcel ID or address, Relio can automatically enrich the property with county assessor data.
</Info>

## Parcel Mapping

Relio uses PostGIS for advanced geospatial features:

- **Point geometry** (`geom`) - Auto-maintained by database trigger from lat/lng
- **Parcel boundaries** (`geomBoundary`) - GeoJSON polygon of property boundaries
- **Spatial queries** - Find properties within radius, polygons, or territories

  View properties on an interactive map with parcel boundaries and clustering
</Card>

## Property Enrichment

Automatic enrichment from multiple sources:

<Steps>
  <Step title="Address Validation">
    Geocode addresses to get accurate lat/lng coordinates (Smarty Streets)
  </Step>
  
  <Step title="Parcel Data">
    Fetch parcel boundaries, APN, and legal descriptions (Regrid)
  </Step>
  
  <Step title="County Assessor">
    Pull tax records, ownership, sale history, and property characteristics
  </Step>
  
  <Step title="Amenity Scores">
    Calculate walk scores, transit access, and nearby amenities
  </Step>
</Steps>

## Multi-Family Properties

For multi-family properties, use these fields:

- `units` - Total unit count
- `propertyType: "multi_family"`
- Store unit mix details in custom fields or related tables
- Use `capRate` for income property analysis

<Tip>
  Track individual unit details using custom fields or a separate unit tracking system for larger properties.
</Tip>

## Property Relationships

Link properties to:
- **Contacts** - Owners, buyers, sellers, agents, tenants
- **Companies** - Property managers, developers, brokerages
- **Deals** - Active transactions
- **Activities** - Showings, inspections, appraisals

## Best Practices

<AccordionGroup>
  <Accordion title="Always provide complete addresses">
    Include street, city, state, and ZIP for accurate geocoding and enrichment. Incomplete addresses may fail to enrich.
  </Accordion>
  
  <Accordion title="Use parcel IDs when available">
    Linking `llUuid` or `parcelId` enables cross-workspace parcel matching and ensures data accuracy.
  </Accordion>
  
  <Accordion title="Track status changes">
    Update `status` as properties move through your pipeline: `active` â†’ `under_contract` â†’ `sold`
  </Accordion>
  
  <Accordion title="Store prices in cents">
    Always convert dollar amounts to cents before storing in price fields to avoid precision errors.
  </Accordion>
  
  <Accordion title="Monitor enrichment freshness">
    Check `lastEnrichedAt` and re-enrich properties periodically to keep assessor data current.
  </Accordion>
</AccordionGroup>
